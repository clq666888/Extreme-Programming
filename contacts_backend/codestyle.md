>Standard source: Google Python Style Guide (PEP8)

## 1. file naming
-Python files use lowercase+underline, for example: app. py, contacts_appi. py
-The file name should express the function as much as possible

## 2. indentation
-Use 4 spaces to indent
-Do not use Tab

## 3. Naming Convention
-Variables and functions: snake_case
-Class name: CapWords
-Constant: all uppercase+underline

## 4. annotation
-Module annotation: Description of module functionality at the top of the file
-Function comment: Use docstring
-Complex logic requires inline comments

## 5. blank line
-Two empty lines between top-level function and class
-Leave a blank line between functions within a class

## 6. Import order
-Standard library → Third party library → Local module
-Leave a blank line between each group

## 7. Line length
-Each line should not exceed 79 characters
